## 5.1 以 Doc 项目为例, 演示如何使用 Git 管理本地项目代码
```
mkdir doc                           // 新建项目
cd doc                              // 进入项目目录
git init                            // 使用 init 命令初始化 doc 目录, 初始化成功后, doc 目录下的所有文件都将交给 git 管理
touch README.md                     // 新建 README.md 文件
echo '# doc' >> README.md           // 向 README.md 文件中追加内容 ''# doc'
cat README.md                       // 查看 README.md 文件的内容
git status                          // 查看当前分支的变更情况
git add README.md                   // 把对于 README.md 文件的修改添加到当前分支的暂存区
git commit -m 'edit README.md'      // 将暂存区中的内容提交到本地代码库( 也叫本地版本库 )
git log -1                          // 查看提交日志, -1 表示查看最新的一条

ssh-keygen -t rsa –C "caffeine.dude.go@gmail.com"                               // 生成自己的 ssh 公私钥, 该文件位于用户主目录的 .ssh 目录下
https://github.com/ > Settings > SSH keys > id_rsa.pub > bai-hasee-pub-key      // 登录 GitHub 将公钥信息存入 SSH keys
Create a new repository > Repository name > doc                                 // 在 GitHub 上新建 doc 项目
git remote add origin git@github.com:snoredude/doc.git                          // 将 GitHub 上的 doc 项目作为 本地 doc 项目的远端
git branch -M main                                                              // 修改分支名称 master 为 main ( 有趣 )
git push -u origin main                                                         // 推送本地的 commit 到远端
```

## 5.3 最小化配置
```
git config --global user.name "snoredude"
git config --global user.email "caffeine.dude.go@gmail.com"
```
--global 是全局级别的配置, 此外, 还有仓库级别的配置 --local, 以及系统级别的配置 --system  
--local 优先级更高, --system 优先级最低  

编辑配置文件:  
```
git config --global --edit
git config --local --edit 
git config --system --edit 
```

## 5.4 代理设置
```
git config --global --list
git config --global http.proxy 127.0.0.1:1080
git config --global https.proxy 127.0.0.1:1080
git config --global http.sslVerify false
git config --global --unset http.proxy
git config --global --unset https.proxy
```
代理一般在翻墙时使用, 不翻墙时需要取消代理  
设置 http.sslVerify = false 表示禁用 Git 的 SSL 验证  

## 5.5 四块区域
1. Workspace: 工作区 ( 项目代码所在的目录 )
2. Index / Stage: 暂存区 ( 位于工作区的 .git 目录中 )
3. Repository: 本地仓库 ( 代码的版本库, 工作区的 .git 目录就是本地仓库, 它包含了项目代码的第一个分支 master, 以及指向 master 的一个指针 HEAD )
4. Remote: 远程仓库

## 5.7 详细命令
### 5.7.1 git commit
git commit 主要是将暂存区里的更改提交到本地的版本库  
每次使用 git commit 都会在本地版本库生成一个 40 位的哈希值, 这个哈希值也叫做 commit-id  
commit-id 在版本回退的时候非常有用, 它相当于一个快照, 可以在未来的任何时候通过与 git reset commit-id 命令回到当次 commit  
示例:  
```
git log -1
git reset 4f78b2b71afeef22f0947603365f445c1d5ccd1b
git restore .
```

`git commit -m 'commit massage'`  
-m 参数指定提交的描述信息

`git commit -a -m 'commit massage'`
-a 参数会把没有保存到暂存区的文件修改也提交掉, 一般不建议使用    

`git commit --amend`
把当前暂存区的修改追加到上次的 commit 中

`git commit --help`
查看帮助  

### 5.7.2 git merge
1. 合并 dev 到当前分支( 该次合并会作为当前分支的一次 commit ): `git merge dev`
2. rebase 操作
```
git pull origin main
git checkout dev
git rebase main
解决冲突
git rebase --continue
git checkout main
git merge dev
```

## 5.8 配置 fork 项目的本地仓库代码和原仓库代码同步
```
原仓库：https://github.com/cncamp/golang.git
fork
clone 到本地
在项目目录下，查看远程仓库路径
git remote -v
origin  https://github.com/snoredude/golang.git (fetch)
origin  https://github.com/snoredude/golang.git (push)
只显示2行内容，说明该项目还未设置upstream （上游代码库）
设置upstream
git remote add upstream https://github.com/cncamp/golang.git
检查是否成功
git remote -v
origin  https://github.com/snoredude/golang.git (fetch)
origin  https://github.com/snoredude/golang.git (push)
upstream        https://github.com/cncamp/golang.git (fetch)
upstream        https://github.com/cncamp/golang.git (push)
检查本地代码是否修改
git status
如果本地有修改，需要先从本地仓库推送到GitHub仓库。
抓取源仓库的更新
git fetch upstream
* [new branch]      master     -> upstream/master
上游仓库的更新（commit）会本存储为本地的分支 upstream/master
切换分支
切换到需要更新合并的分支
git checkout master
如果本身就位于该分支，不需要切换
执行合并
git merge upstream/master
如有代码冲突，需要解决。
推送到Gibhub
git push origin maste
```
