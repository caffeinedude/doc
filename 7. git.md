## 7.3 基本配置
```
git config --global user.name "snoredude"
git config --global user.email "caffeine.dude.go@gmail.com"
```
--global 是全局级别的配置, 此外, 还有仓库级别的配置 --local, 以及系统级别的配置 --system  
--local 优先级更高, --system 优先级最低  

编辑配置文件:  
```
git config --global --edit
git config --local --edit 
git config --system --edit 
```

## 7.4 代理设置
```
git config --global --list
git config --global http.proxy 127.0.0.1:8087
git config --global http.sslVerify false
git config --global --unset http.proxy
```
代理一般在翻墙时使用, 不翻墙时需要取消代理  
设置 http.sslVerify = false 表示禁用 Git 的 SSL 验证  

## 7.5 四块区域
1. Workspace: 工作区 ( 项目代码所在的目录 )
2. Index / Stage: 暂存区 ( 位于工作区的 .git 目录中 )
3. Repository: 本地仓库 ( 代码的版本库, 工作区的 .git 目录就是本地仓库, 它包含了项目代码的第一个分支 master, 以及指向 master 的一个指针 HEAD )
4. Remote: 远程仓库

## 7.6 基本命令
1. 修改
2. 放弃修改: `git restore filename`
3. 暂存: `git add filename`
4. 撤销暂存: `git restore --staged filename`
5. 提交: `git commit -m "description"`
6. 推送: `git push origin main`
- - -
7. 查看仓库变更情况: `git status`


8. 添加单个文件:  `git add filename`
9. 添加当前目录下的所有文件: `git add .`
10. 添加仓库中的所有文件: `git add -A`

11. 查看所在分支( 当前分支名称前会出现 * 号 ): `git branch`
12. 查看所有分支( 包括本地和远端 ): `git branch -av`
13. 创建 dev 分支: `git branch dev`
14. 切换到 dev 分支: `git checkout dev`
15. 创建并切换到新的 issue 分支: `git checkout -b issue`
16. 删除 issue 分支: `git branch -d issue `

17. 查看 commit 日志: `git log`
18. 查看最近 3 条 commit 日志: `git log -3`
19. 查看指定文件的 commit 日志: `git log filename`

20. 给 commit 打 tag: `git tag mytag commitid`

## 7.12 git merge  
1. 合并 dev 到当前分支( 该次合并会作为当前分支的一次 commit ): `git merge dev`
2. rebase 操作
```
git pull origin main
git checkout dev
git rebase main
解决冲突
git rebase --continue
git checkout main
git merge dev
```

## 7.13 fork 本地仓库代码和原仓库代码同步
```
原仓库：https://github.com/cncamp/golang.git
fork
clone 到本地
在项目目录下，查看远程仓库路径
git remote -v
origin  https://github.com/snoredude/golang.git (fetch)
origin  https://github.com/snoredude/golang.git (push)
只显示2行内容，说明该项目还未设置upstream （上游代码库）
设置upstream
git remote add upstream https://github.com/cncamp/golang.git
检查是否成功
git remote -v
origin  https://github.com/snoredude/golang.git (fetch)
origin  https://github.com/snoredude/golang.git (push)
upstream        https://github.com/cncamp/golang.git (fetch)
upstream        https://github.com/cncamp/golang.git (push)
检查本地代码是否修改
git status
如果本地有修改，需要先从本地仓库推送到GitHub仓库。
抓取源仓库的更新
git fetch upstream
* [new branch]      master     -> upstream/master
上游仓库的更新（commit）会本存储为本地的分支 upstream/master
切换分支
切换到需要更新合并的分支
git checkout master
如果本身就位于该分支，不需要切换
执行合并
git merge upstream/master
如有代码冲突，需要解决。
推送到Gibhub
git push origin maste
```

### 7.15 本地
```
git fetch remote 拉取远端的所有分支和变更  
git pull remote test 拉取远端的某个分支的变更，然后 merge 到本地分支
git push remote名称 test 推送本地分支到远端
```
