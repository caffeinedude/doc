## 关于表单（3 张表）
**数据库表：form_field**  
对应的结构体是：FormField  
该表中存放的是活动的表单具体信息（每一条表单数据）  
在 "平安问答" 模块下，每一个题目都是一个活动，都有一张对应的表单  
以我新建的一个题目："单选题" 为例   
四个选项会在 form_field 表里存四条数据，字段解析如下：  
```
id          // uint64，自增的唯一id
created_at  // 选项的创建时间
updated_at  // 选项的修改时间
deleted_at  // 选项的删除时间
index       // int，选项的索引值（0，1，2，3...），表示该选项在第几行（index = 0 表示该选项在第一行）
name        // string，选项的内容
required    // bool，为 1 表示是正确选项，为 0 表示是错误选项
type        // int，表示该条选项（记录）的类型，0 文本输入，1 数字输入，2 图片上传，3 是否选择，4 单项选择，5 多项选择，6 下拉选择，7 日期选择，8 时间选择，9 城市选择，10 滚动选择，11 打分评价，【在 题目表单 里似乎不需要使用这个字段】
placeholder // string，在 题目表单 里表示的含义是该选项的解析和说明信息
options     // string，在 题目表单 里没有使用
owner_id    // uint64，在 题目表单 表示属于哪一个题目
owner_type  // string，在 题目表单 表示属于哪一个活动，在这里所有的选项都属于活动 "item"
```

**数据库表：form_value**  
对应的结构体是：FormValue  
该表中存放的是用户在 活动表单 中输入的数据  
在 "平安问答" 模块下，一道题一张表单，也就是用户对这道题填写的答案  
表的字段解析如下：
```
id              // uint64，自增的唯一id
created_at      // 选项的创建时间
updated_at      // 选项的修改时间
deleted_at      // 选项的删除时间
form_field_id   // form_field 表中选项的id，标识该条记录是哪个选项的答案
owner_id        // uint64，表示属于哪一个题目
owner_type      // string，在这张表里表示属于哪一个活动 "form_submit_record"
value           // string，用户输入的值
```  

**数据库表：form_submit_record**  
对应的结构体是：FormSubmitRecord  
该表中存放的是 用户的表单提交记录  
表的字段解析如下：
```
id              // uint64，自增的唯一id
created_at      // 选项的创建时间
updated_at      // 选项的修改时间
deleted_at      // 选项的删除时间
owner_id        // uint64，表示属于哪一个题目
owner_type      // string，在这张表里表示属于哪一个活动 "item"
user_id         // uint64，表示是哪个用户提交的表单
```
